**파일 경로:** `backend/app/api/uploads.py`
**라우터 prefix:** `/api/uploads`
**최종 수정일:** 2026-03-24

---

## 엔드포인트 목록

| 메서드 | 경로 | 설명 | 권한 |
|---|---|---|---|
| `POST` | `/api/uploads/image` | 이미지 파일 업로드 | Bearer JWT |
| `GET` | `/uploads/{path}` | 업로드 파일 서빙 | 없음 (공개) |

## 주요 엔드포인트 상세

### `POST /api/uploads/image`

**설명:** 이미지 파일을 서버에 업로드. 아래 순서로 검증 후 저장:

1. `Content-Type` MIME 타입 확인 (`settings.allowed_image_types`)
2. 파일 전체 읽기 후 크기 검사 (`settings.max_upload_size`)
3. 파일 헤더 12바이트로 매직 바이트 검증 (확장자 위변조 방지)
4. UUID 기반 고유 파일명 생성 후 `settings.upload_dir`에 저장

**인증:** Bearer JWT 필요

**요청 형식:** `multipart/form-data`

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `file` | UploadFile | ✓ | 업로드할 이미지 파일 |

**응답 (200):**
```json
{
  "url": "/uploads/a3f2c1d4e5b6a7c8d9e0f1a2b3c4d5e6.jpg"
}
```

**에러:**
- `400`: 허용되지 않은 MIME 타입
- `400`: 파일 크기 초과
- `400`: 파일 크기가 12바이트 미만 (유효하지 않은 이미지)
- `400`: 매직 바이트가 이미지 형식과 불일치 (확장자 위변조 의심)

---

### `GET /uploads/{path}`

**설명:** 업로드된 파일 직접 서빙. 인증 없이 접근 가능 (프로필 이미지 등 공개 자원). 경로 순회 공격(`../`) 방지 처리 포함.

**인증:** 불필요

**경로 파라미터:**
| 파라미터 | 설명 |
|---|---|
| `path` | 파일명 (예: `a3f2c1d4...jpg`) |

**응답:** 파일 바이너리 (`FileResponse`)

**에러:**
- `403`: 경로 순회 공격 감지
- `404`: 파일 없음

**비고:** 이 엔드포인트는 `main.py`에 직접 등록되며 `uploads` 라우터와 별개.

## 지원 이미지 포맷 및 매직 바이트

| 포맷 | 매직 바이트 |
|---|---|
| JPEG | `\xFF\xD8\xFF` |
| PNG | `\x89PNG` |
| GIF | `GIF8` |
| WebP | `RIFF....WEBP` (4+8번째 바이트) |
